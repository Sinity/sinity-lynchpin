# Instrumentation Metadata Plan

## Scope
Prepare sinity-analysis to ingest metadata generated by instrumentation owned in `/realm/sinnix` (asciinema, audio, screencaps, telemetry) without duplicating raw binaries.

## Data Contracts
| Stream | Source Path | Expected Artefact | Metadata Fields |
|--------|-------------|-------------------|-----------------|
| Terminal (asciinema) | `/realm/data/asciinema_recording/` | `.cast` per session/hour | filename, start/end UTC, shell, terminal, size |
| Audio (PipeWire) | `/realm/data/audio/raw/` | Hourly FLAC/Opus chunks | filename, channels, sample rate, duration, checksum |
| Screen Capture | `/realm/data/screenshot/` (planned) | PNG/JPEG or video segments | filename, resolution, colour profile, window id (optional) |
| System Telemetry | TBD (`/realm/data/telemetry/…`) | NDJSON/CSV metrics | timestamp, host, metric name, value, units |

## Pipeline Outline
1. **Metadata harvest scripts** (to add under `scripts/`):
   - `collect_asciinema_metadata.py`: walk newest `.cast` files, emit NDJSON to `data/derived/asciinema_metadata.jsonl`.
   - `collect_audio_metadata.py`: ffprobe-based duration/channel extraction, writes to `data/derived/audio_metadata.jsonl`.
   - `collect_screen_metadata.py`: parse filenames & optional AW annotations once capture stabilises.
   - `collect_telemetry_metadata.py`: convert telemetry dumps into unified schema.

2. **Integration with dashboards**: join metadata (esp. terminal/audio) with `activity_timeline` using timestamp overlap → surface instrumentation coverage vs focus metrics.

3. **Sinevec embeddings**: once metadata is stable, embed transcripts/descriptions (`sinevec embed-artifacts --category instrumentation`).

4. **Docs & Ownership**: keep configuration notes in `/realm/sinnix/docs/breakthrough.md`; mirror ingestion details here for analytics consumers.
